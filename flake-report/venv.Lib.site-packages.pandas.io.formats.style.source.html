122  
123 Parameters
124 ----------
125 data : Series or DataFrame
126 Data to be styled - either a Series or DataFrame.
127 precision : int, optional
128 Precision to round floats to. If not given defaults to

130  
131 .. versionchanged:: 1.4.0
132 table_styles : list-like, default None
133 List of {selector: (attr, value)} dicts; see Notes.
134 uuid : str, default None
135 A unique identifier to avoid CSS collisions; generated automatically.
136 caption : str, tuple, default None
137 String caption to attach to the table. Tuple only used for LaTeX dual captions.
138 table_attributes : str, default None
139 Items that show up in the opening ``<table>`` tag
140 in addition to automatic (by default) id.
141 cell_ids : bool, default True
142 If True, each cell will have an ``id`` attribute in their HTML tag.
143 The ``id`` takes the form ``T_<uuid>_row<num_row>_col<num_col>``
144 where ``<uuid>`` is the unique identifier, ``<num_row>`` is the row
145 number and ``<num_col>`` is the column number.
146 na_rep : str, optional
147 Representation for missing values.
148 If ``na_rep`` is None, no special formatting is applied, and falls back to
149 ``pandas.options.styler.format.na_rep``.
150  
151 uuid_len : int, default 5
152 If ``uuid`` is not specified, the length of the ``uuid`` to randomly generate

154  
155 .. versionadded:: 1.2.0
156  
157 decimal : str, optional
158 Character used as decimal separator for floats, complex and integers. If not
159 given uses ``pandas.options.styler.format.decimal``.
160  
161 .. versionadded:: 1.3.0
162  
163 thousands : str, optional, default None
164 Character used as thousands separator for floats, complex and integers. If not
165 given uses ``pandas.options.styler.format.thousands``.
166  
167 .. versionadded:: 1.3.0
168  
169 escape : str, optional
170 Use 'html' to replace the characters ``&``, ``<``, ``>``, ``'``, and ``"``
171 in cell display string with HTML-safe sequences.
172 Use 'latex' to replace the characters ``&``, ``%``, ``$``, ``#``, ``_``,
173 ``{``, ``}``, ``~``, ``^``, and ``\`` in the cell display string with
174 LaTeX-safe sequences. If not given uses ``pandas.options.styler.format.escape``.
175  
176 .. versionadded:: 1.3.0
177 formatter : str, callable, dict, optional
178 Object to define how values are displayed. See ``Styler.format``. If not given
179 uses ``pandas.options.styler.format.formatter``.
180  
181 .. versionadded:: 1.4.0
182  
183 Attributes
184 ----------
185 env : Jinja2 jinja2.Environment
186 template_html : Jinja2 Template
187 template_html_table : Jinja2 Template
188 template_html_style : Jinja2 Template
189 template_latex : Jinja2 Template

196  
197 Notes
198 -----
199 Most styling will be done by passing style functions into
200 ``Styler.apply`` or ``Styler.applymap``. Style functions should
201 return values with strings containing CSS ``'attr: value'`` that will
202 be applied to the indicated cells.
203

207

211 where `k` is its level in a MultiIndex
212 * Index label cells include
213

276 def concat(self, other: Styler) -> Styler:
277 """
278 Append another Styler to combine the output into a single table.
279  
280 .. versionadded:: 1.5.0
281  
282 Parameters
283 ----------
284 other : Styler
285 The other Styler object which has already been styled and formatted. The
286 data for this Styler must have the same columns as the original, and the
287 number of index levels must also be the same to render correctly.
288  
289 Returns
290 -------
291 Styler
292  
293 Notes
294 -----
295 The purpose of this method is to extend existing styled dataframes with other
296 metrics that may be useful but may not conform to the original's structure.
297 For example adding a sub total row, or displaying metrics such as means,
298 variance or counts.
299  
300 Styles that are applied using the ``apply``, ``applymap``, ``apply_index``
301 and ``applymap_index``, and formatting applied with ``format`` and
302 ``format_index`` will be preserved.
303  
304 .. warning::
305 Only the output methods ``to_html``, ``to_string`` and ``to_latex``
306 currently work with concatenated Stylers.
307  
308 Other output methods, including ``to_excel``, **do not** work with
309 concatenated Stylers.
310  
311 The following should be noted:
312  
313 - ``table_styles``, ``table_attributes``, ``caption`` and ``uuid`` are all
314 inherited from the original Styler and not ``other``.
315 - hidden columns and hidden index levels will be inherited from the
316 original Styler
317 - ``css`` will be inherited from the original Styler, and the value of
318 keys ``data``, ``row_heading`` and ``row`` will be prepended with
319 ``foot0_``. If more concats are chained, their styles will be prepended
320 with ``foot1_``, ''foot_2'', etc., and if a concatenated style have
321 another concatanated style, the second style will be prepended with
322 ``foot{parent}_foot{child}_``.
323  
324 A common use case is to concatenate user defined functions with
325 ``DataFrame.agg`` or with described statistics via ``DataFrame.describe``.
326 See examples.
327  
328 Examples
329 --------
330 A common use case is adding totals rows, or otherwise, via methods calculated
331 in ``DataFrame.agg``.
332  
333 >>> df = DataFrame([[4, 6], [1, 9], [3, 4], [5, 5], [9,6]],
334 ... columns=["Mike", "Jim"],
335 ... index=["Mon", "Tue", "Wed", "Thurs", "Fri"])
336 >>> styler = df.style.concat(df.agg(["sum"]).style) # doctest: +SKIP
337  
338 .. figure:: ../../_static/style/footer_simple.png
339  
340 Since the concatenated object is a Styler the existing functionality can be
341 used to conditionally format it as well as the original.
342  
343 >>> descriptors = df.agg(["sum", "mean", lambda s: s.dtype])
344 >>> descriptors.index = ["Total", "Average", "dtype"]
345 >>> other = (descriptors.style
346 ... .highlight_max(axis=1, subset=(["Total", "Average"], slice(None)))
347 ... .format(subset=("Average", slice(None)), precision=2, decimal=",")
348 ... .applymap(lambda v: "font-weight: bold;"))
349 >>> styler = (df.style
350 ... .highlight_max(color="salmon")
351 ... .set_table_styles([{"selector": ".foot_row0",
352 ... "props": "border-top: 1px solid black;"}]))
353 >>> styler.concat(other) # doctest: +SKIP
354  
355 .. figure:: ../../_static/style/footer_extended.png
356  
357 When ``other`` has fewer index levels than the original Styler it is possible
358 to extend the index in ``other``, with placeholder levels.
359  
360 >>> df = DataFrame([[1], [2]], index=pd.MultiIndex.from_product([[0], [1, 2]]))
361 >>> descriptors = df.agg(["sum"])
362 >>> descriptors.index = pd.MultiIndex.from_product([[""], descriptors.index])
363 >>> df.style.concat(descriptors.style) # doctest: +SKIP
364 """
365 if not isinstance(other, Styler):
366 raise TypeError("`other` must be of type `Styler`")
367 if not self.data.columns.equals(other.data.columns):
368 raise ValueError("`other.data` must have same columns as `Styler.data`")
369 if not self.data.index.nlevels == other.data.index.nlevels:
370 raise ValueError(
371 "number of index levels must be same in `other` "
372 "as in `Styler`. See documentation for suggestions."
373 )
374 self.concatenated.append(other)
375 return self

391 def set_tooltips(
392 self,
393 ttips: DataFrame,
394 props: CSSProperties | None = None,
395 css_class: str | None = None,
396 ) -> Styler:
397 """
398 Set the DataFrame of strings on ``Styler`` generating ``:hover`` tooltips.
399  
400 These string based tooltips are only applicable to ``<td>`` HTML elements,
401 and cannot be used for column or index headers.
402  
403 .. versionadded:: 1.3.0
404  
405 Parameters
406 ----------
407 ttips : DataFrame
408 DataFrame containing strings that will be translated to tooltips, mapped
409 by identical column and index values that must exist on the underlying
410 Styler data. None, NaN values, and empty strings will be ignored and
411 not affect the rendered HTML.
412 props : list-like or str, optional
413 List of (attr, value) tuples or a valid CSS string. If ``None`` adopts
414 the internal default values described in notes.
415 css_class : str, optional
416 Name of the tooltip class used in CSS, should conform to HTML standards.
417 Only useful if integrating tooltips with external CSS. If ``None`` uses the
418 internal default value 'pd-t'.
419  
420 Returns
421 -------
422 Styler
423  
424 Notes
425 -----
426 Tooltips are created by adding `<span class="pd-t"></span>` to each data cell
427 and then manipulating the table level CSS to attach pseudo hover and pseudo
428 after selectors to produce the required the results.
429  
430 The default properties for the tooltip CSS class are:
431  
432 - visibility: hidden
433 - position: absolute
434 - z-index: 1
435 - background-color: black
436 - color: white
437 - transform: translate(-20px, -20px)
438  
439 The property 'visibility: hidden;' is a key prerequisite to the hover
440 functionality, and should always be included in any manual properties
441 specification, using the ``props`` argument.
442  
443 Tooltips are not designed to be efficient, and can add large amounts of
444 additional HTML for larger tables, since they also require that ``cell_ids``
445 is forced to `True`.
446  
447 Examples
448 --------
449 Basic application
450  
451 >>> df = pd.DataFrame(data=[[0, 1], [2, 3]])
452 >>> ttips = pd.DataFrame(
453 ... data=[["Min", ""], [np.nan, "Max"]], columns=df.columns, index=df.index
454 ... )
455 >>> s = df.style.set_tooltips(ttips).to_html()
456  
457 Optionally controlling the tooltip visual display
458  
459 >>> df.style.set_tooltips(ttips, css_class='tt-add', props=[
460 ... ('visibility', 'hidden'),
461 ... ('position', 'absolute'),
462 ... ('z-index', 1)]) # doctest: +SKIP
463 >>> df.style.set_tooltips(ttips, css_class='tt-add',
464 ... props='visibility:hidden; position:absolute; z-index:1;')
465 ... # doctest: +SKIP
466 """
467 if not self.cell_ids:
468 # tooltips not optimised for individual cell check. requires reasonable
469 # redesign and more extensive code for a feature that might be rarely used.
470 raise NotImplementedError(
471 "Tooltips can only render with 'cell_ids' is True."
472 )
473 if not ttips.index.is_unique or not ttips.columns.is_unique:
474 raise KeyError(
475 "Tooltips render only if `ttips` has unique index and columns."
476 )
477 if self.tooltips is None: # create a default instance if necessary
478 self.tooltips = Tooltips()
479 self.tooltips.tt_data = ttips
480 if props:
481 self.tooltips.class_properties = props
482 if css_class:
483 self.tooltips.class_name = css_class
484  
485 return self

739 **Cell Styles**
740  
741 LaTeX styling can only be rendered if the accompanying styling functions have
742 been constructed with appropriate LaTeX commands. All styling
743 functionality is built around the concept of a CSS ``(<attribute>, <value>)``
744 pair (see `Table Visualization <../../user_guide/style.ipynb>`_), and this
745 should be replaced by a LaTeX
746 ``(<command>, <options>)`` approach. Each cell will be styled individually
747 using nested LaTeX commands with their accompanied options.
748  
749 For example the following code will highlight and bold a cell in HTML-CSS:
750  
751 >>> df = pd.DataFrame([[1,2], [3,4]])
752 >>> s = df.style.highlight_max(axis=None,
753 ... props='background-color:red; font-weight:bold;')
754 >>> s.to_html() # doctest: +SKIP
755  
756 The equivalent using LaTeX only commands is the following:
757  
758 >>> s = df.style.highlight_max(axis=None,
759 ... props='cellcolor:{red}; bfseries: ;')
760 >>> s.to_latex() # doctest: +SKIP
761  
762 Internally these structured LaTeX ``(<command>, <options>)`` pairs
763 are translated to the
764 ``display_value`` with the default structure:
765 ``\<command><options> <display_value>``.
766 Where there are multiple commands the latter is nested recursively, so that
767 the above example highlighted cell is rendered as
768 ``\cellcolor{red} \bfseries 4``.
769  
770 Occasionally this format does not suit the applied command, or
771 combination of LaTeX packages that is in use, so additional flags can be
772 added to the ``<options>``, within the tuple, to result in different
773 positions of required braces (the **default** being the same as ``--nowrap``):
774  
775 =================================== ============================================
776 Tuple Format Output Structure
777 =================================== ============================================
778 (<command>,<options>) \\<command><options> <display_value>
779 (<command>,<options> ``--nowrap``) \\<command><options> <display_value>
780 (<command>,<options> ``--rwrap``) \\<command><options>{<display_value>}
781 (<command>,<options> ``--wrap``) {\\<command><options> <display_value>}
782 (<command>,<options> ``--lwrap``) {\\<command><options>} <display_value>
783 (<command>,<options> ``--dwrap``) {\\<command><options>}{<display_value>}
784 =================================== ============================================
785  
786 For example the `textbf` command for font-weight
787 should always be used with `--rwrap` so ``('textbf', '--rwrap')`` will render a
788 working cell, wrapped with braces, as ``\textbf{<display_value>}``.
789  
790 A more comprehensive example is as follows:
791  
792 >>> df = pd.DataFrame([[1, 2.2, "dogs"], [3, 4.4, "cats"], [2, 6.6, "cows"]],
793 ... index=["ix1", "ix2", "ix3"],
794 ... columns=["Integers", "Floats", "Strings"])
795 >>> s = df.style.highlight_max(
796 ... props='cellcolor:[HTML]{FFFF00}; color:{red};'
797 ... 'textit:--rwrap; textbf:--rwrap;'
798 ... )
799 >>> s.to_latex() # doctest: +SKIP
800  
801 .. figure:: ../../_static/style/latex_1.png

803 **Table Styles**
804  
805 Internally Styler uses its ``table_styles`` object to parse the
806 ``column_format``, ``position``, ``position_float``, and ``label``
807 input arguments. These arguments are added to table styles in the format:
808  
809 .. code-block:: python
810  
811 set_table_styles([
812 {"selector": "column_format", "props": f":{column_format};"},
813 {"selector": "position", "props": f":{position};"},
814 {"selector": "position_float", "props": f":{position_float};"},
815 {"selector": "label", "props": f":{{{label.replace(':','§')}}};"}
816 ], overwrite=False)
817  
818 Exception is made for the ``hrules`` argument which, in fact, controls all three
819 commands: ``toprule``, ``bottomrule`` and ``midrule`` simultaneously. Instead of
820 setting ``hrules`` to ``True``, it is also possible to set each
821 individual rule definition, by manually setting the ``table_styles``,
822 for example below we set a regular ``toprule``, set an ``hline`` for
823 ``bottomrule`` and exclude the ``midrule``:
824  
825 .. code-block:: python
826  
827 set_table_styles([
828 {'selector': 'toprule', 'props': ':toprule;'},
829 {'selector': 'bottomrule', 'props': ':hline;'},
830 ], overwrite=False)
831  
832 If other ``commands`` are added to table styles they will be detected, and
833 positioned immediately above the '\\begin{tabular}' command. For example to
834 add odd and even row coloring, from the {colortbl} package, in format
835 ``\rowcolors{1}{pink}{red}``, use:
836  
837 .. code-block:: python
838  
839 set_table_styles([
840 {'selector': 'rowcolors', 'props': ':{1}{pink}{red};'}
841 ], overwrite=False)
842  
843 A more comprehensive example using these arguments is as follows:
844  
845 >>> df.columns = pd.MultiIndex.from_tuples([
846 ... ("Numeric", "Integers"),
847 ... ("Numeric", "Floats"),
848 ... ("Non-Numeric", "Strings")
849 ... ])
850 >>> df.index = pd.MultiIndex.from_tuples([
851 ... ("L0", "ix1"), ("L0", "ix2"), ("L1", "ix3")
852 ... ])
853 >>> s = df.style.highlight_max(
854 ... props='cellcolor:[HTML]{FFFF00}; color:{red}; itshape:; bfseries:;'
855 ... )
856 >>> s.to_latex(
857 ... column_format="rrrrr", position="h", position_float="centering",
858 ... hrules=True, label="table:5", caption="Styled LaTeX Table",
859 ... multirow_align="t", multicol_align="r"
860 ... ) # doctest: +SKIP
